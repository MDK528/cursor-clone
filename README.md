# Cursor Clone

A pixel-close static clone of [cursor.com](https://cursor.com), built from scratch using only HTML and CSS as part of a frontend assignment. No frameworks, no JavaScript, no shortcuts.

## What I Built

This project is a desktop-only recreation of the Cursor homepage. Every section was hand-coded by studying the original site's layout, spacing, typography, and color choices.

### Sections Included

1. **Top Navigation Bar** — Logo, nav links, Sign In and Download buttons
2. **Hero Section** — Headline, CTA button, full-width product screenshot
3. **Trusted By** — Row of 8 company logos (Stripe, OpenAI, Linear, Datadog, NVIDIA, Figma, Ramp, Adobe)
4. **Feature Sections (×4)** — Alternating two-column layouts showcasing Agents, Cloud Agents, Surfaces, and Tab Autocomplete
5. **Testimonials** — 6 quote cards in a 3-column grid with avatar, name, and role
6. **Use Cases** — 3-column feature cards with images and descriptions
7. **Changelog** — 4-column grid of recent release entries with version badges and dates
8. **About / Team** — Two-column layout with a team photo and CTA
9. **Recent Blog Highlights** — 4-column grid of blog post cards with author info
10. **Final CTA** — Large centered heading with a Download button
11. **Footer** — 5-column link grid with copyright bar, theme toggle, and language selector

## Constraints

This project was built under the following rules:

- HTML and CSS only — no JavaScript
- No CSS frameworks (no Tailwind, no Bootstrap)
- No animations or transitions
- Desktop layout only — no responsiveness required
- Images and logos as close to the original site as possible

---

## What I Learned

- How to structure a multi-section landing page cleanly using semantic HTML
- Practical use of CSS Flexbox and Grid for different layout patterns
- How to read and replicate a real-world design's spacing, typography, and color system
- Writing meaningful, readable CSS class names instead of generic utility names
